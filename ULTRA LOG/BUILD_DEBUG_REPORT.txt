[3.2] intrin.h NOT FOUND — RESOLVED
────────────────────────────────────
- Root cause: c4core's fast_float_all.h line 221-223 has:
    #if ((defined(_WIN32) || defined(_WIN64)) && !defined(__clang__)) || \
        (defined(_M_ARM64) && !defined(__MINGW32__))
    #include <intrin.h>
  The _M_ARM64 condition is TRUE on iOS cross-compile (from -D_M_ARM64=1),
  but <intrin.h> is Windows-only and doesn't exist in iOS SDK.
  __clang__ is also defined, so the first part of the OR is FALSE,
  but _M_ARM64 && !__MINGW32__ is TRUE, triggering the include.
- This is a c4core bug: _M_ARM64 alone doesn't mean Windows. Should include
  __APPLE__ in the exclusion.
- The CMAKE_CROSSCOMPILING variable is NOT set by the ios.toolchain.cmake,
  so c4core can't detect cross-compile via that path.
- Fix: sed patch in workflow after submodule init:
    sed -i '' 's/defined(_M_ARM64) && !defined(__MINGW32__)/defined(_M_ARM64) \&\& !defined(__MINGW32__) \&\& !defined(__APPLE__)/' \
      pcsx2/3rdparty/ryml/ext/c4core/src/c4/ext/fast_float_all.h
